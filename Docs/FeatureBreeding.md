# INCUBATOR ZERO — Спека: награды за рейды + бридинг (v0.1)

> Цель: одна система наград должна **кормить все механики** (прогресс, бридинг, еда/крафт, косметика), но не раздувать экономику и не превращать игру в “лотерею ради силы”.

---

## 0) Принципы (не обсуждается, иначе всё развалится)

- **Сила ≠ редкая случайность.** Редкость/долгий гринд — в косметике/коллекции. Боевые вещи — максимально контролируемы.
- **Бридинг 2→1**, понятные правила наследования, UI показывает исходы/шансы.
- Всё, что влияет на **случайные результаты**, не продаётся за Robux и не бустится “платой за удачу”.
- Система должна поддерживать **короткие сессии**: 2–4 рейда → понятный прогресс → 1–2 решения в инкубаторе.

---

## 1) Награды за рейды

### 1.1 Типы наград (минимальный набор)

| Награда | Тип | Зачем нужна | Куда тратится (sinks) |
|---|---|---|---|
| **Credits** | soft currency | универсальный клей экономики | апгрейды, комиссии станций, крафт |
| **Biomass** | craft resource | еда/поддержка питомца, мягкий прогресс | крафт еды/кормов, восстановление фертильности |
| **Genome** | meta currency | “плата за генетику” | бридинг, splice-операции, перенос/фиксация |
| **Catalyst Shards: Bio** | craft (combat) | путь к боевым улучшениям | крафт Bio Serum / детерминированных инъекций |
| **Catalyst Shards: Style** | craft (cosmetic) | путь к косметике/трейтом | крафт Style Serum / визуальных трейтов |
| **Core Sample (Boss)** | rare ingredient | ключ к “контролю” без казино | splice, детерминированные комбо/инъекции |

> Если добавишь ещё 2–3 валюты “потому что красиво” — игрок перестанет понимать, зачем он вообще ходит в рейды.

---

### 1.2 Режимы и профили дропа

#### Wave Arena (часто, быстро)
- **Всегда:** Credits + Biomass
- **Часто:** немного Genome
- **Иногда:** Style Shards (очень мало), чтобы поддерживать косметический прогресс без обесценивания боссов

**Роль режима:** быстрый фарм “жизни” (еда/поддержка/базовый прогресс), не “путь к имбе”.

#### Boss Contract (реже, но “значимо”)
- **Всегда:** Credits + Genome
- **Часто:** Bio Shards
- **Редко / гарант по правилам:** Core Sample
- **Иногда:** Style Shards (мало)

**Роль режима:** генетика/боевой прогресс через контроль и ингредиенты.

#### Daily / Weekly Contracts (возврат в игру)
- Dailies: Credits + Style Shards (основной источник “визуальной охоты”)
- Weeklies: Style Shards + уникальные Style-компоненты / гарант 1 “косметического ключа”

**Роль:** ретеншн без усиления P2W-ощущения.

---

### 1.3 Модификаторы рейдов → смещение наград (не “шанс удачи”)

Модификаторы должны:
- менять **состав дропа** (например +Bio Shards, -Biomass),
- менять **условия боя** (resistances / hazards),
- но **не** давать “+X% к шансу редкой мутации”.

Примеры:
- `Toxic Bloom`: +Bio Shards, враги оставляют токсичные зоны
- `Neon Leak`: +Style Shards, световые ловушки/дебафф видимости
- `Famine`: +Genome, -Biomass, но падает “заменитель еды”

## 2) Преобразования и крафт (цепочка из двух шагов)

### 2.1 Рафинирование (Shard → Serum)

- **10 Catalyst Shards (Bio/Style) → 1 Serum (Bio/Style)**
- Комиссия: **Credits**

> Это твой контролируемый “гринд-слой”: игрок понимает, что 10→1, и не ждёт чудес.

### 2.2 Serum → продукты (Dose / Injection)

**Dose** — расходник, который используется ТОЛЬКО для “операций с геномом”, где есть случайность.  
**Injection** — детерминированный результат (заданная мутация/фикс).

- **Genome Dose (Bio)**: `1 Bio Serum + Genome + Credits`  
  Использование: операции с боевыми мутациями / сложные splice-процессы.
- **Trait Dose (Style)**: `1 Style Serum + Genome + Credits`  
  Использование: операции с косметическими трейтами / визуальные splice-комбо.

- **Deterministic Injection (Bio/Style)**: `Serum + Core Sample + Credits (+ Genome)`  
  Использование: детерминированное добавление/замена *конкретной* мутации/трейта.

---

## 3) Бридинг (2→1) — механика и правила

### 3.1 Термины (чтобы не путаться)

- **Dominant Gene** — определяет “скелет” ребёнка (вид/архетип/роль/база).
- **Cosmetic Traits** — визуальные черты (ауры, паттерны, свечение). Не дают силы.
- **Combat Mutations** — боевые мутации (2 слота в MVP).
- **Strain (напряжение генома)** — мера “сложности сборки” питомца.
- **Fertility Charges** — сколько раз питомец может участвовать в бридинге без восстановления.

---

### 3.2 Поток игрока (UX)

1) Выбор Parent A + Parent B
2) Экран “Потенциальные исходы” (ОБЯЗАТЕЛЬНО):
    - доминантность (кто определяет базу ребёнка)
    - какие Combat Mutations могут унаследоваться (по слотам) + шансы
    - какие Cosmetic Traits могут унаследоваться + шансы
    - итоговая стоимость + **почему** такая (Strain)
3) Выбор режима:
    - **Standard Breed** (обычный)
    - **Splice** (контролируемый, дороже, требует Core Sample)
4) Инкубация (таймер короткий/средний, чтобы игрок видел “процесс”, но не уходил навсегда)
5) Рождение ребёнка → опциональные пост-операции (детерминированные/ресурсные)

---

### 3.3 Standard Breed (без уничтожения родителей)

**Родители остаются**, но:
- тратят **1 Fertility Charge**
- получают **cooldown** (чтобы не спамить)

**Стоимость:**
- `Credits + Genome + Fee(Strain)`
- (опционально) `Style Dose` если включён “Wild Cosmetic Roll”
- (опционально) `Bio Dose` если включён “Wild Combat Roll” (НЕ рекомендую для MVP)

---

### 3.4 Splice (контроль вместо казино)

Splice = режим, где игрок **платит больше**, но получает **гарант** по одному параметру.

**Требует:** `Core Sample + Genome + Credits` (+ Strain fee)

Выбор гаранта (только один на операцию):
- гарантировать **конкретный Cosmetic Trait**
- гарантировать **конкретную Combat Mutation в слоте 1 или 2**
- гарантировать **Dominant Gene** (если игрок хочет выбрать доминанта вручную)

> Splice — твой главный “анти-фрустрационный” инструмент. Он дорогой, но честный.

---

## 4) Наследование: простые правила (игрок должен запомнить за 30 секунд)

### 4.1 Dominant Gene

Вариант для MVP (самый простой):
- Доминант по умолчанию = **более редкий / более “прокачанный”** родитель.
- Альтернатива: выбрать доминанта можно через **Splice-гарант**.

---

### 4.2 Cosmetic Traits (визуальные)

Пусть у каждого питомца будет, например, **0–3 слота Cosmetic Traits** (MVP можно 0–2).

Правила:
1) Если **оба родителя** имеют **одинаковый trait** → **100% наследуется**
2) Если trait только у **одного** родителя → наследование с шансом `P_single` (например 40%)
3) Если у родителей **разные traits в одном слоте** → выбор:
    - trait A: `P_A` (например 45%)
    - trait B: `P_B` (например 45%)
    - пусто: `P_none` (например 10%)

Опционально (для ретеншна, но не обязательно в MVP):
- **Wild Cosmetic Roll**: небольшой шанс получить **новый** cosmetic trait (не силу!)
    - включается за `Trait Dose`
    - шанс фиксированный и показывается в UI
    - не стакается бесконечно

---

### 4.3 Combat Mutations (боевые), 2 слота в MVP

Правила по каждому слоту отдельно:

1) Если у обоих родителей в этом слоте **одинаковая мутация** → **100% у ребёнка**
2) Если у обоих есть мутация, но **разная** → `50/50` (или `55/45` в пользу доминанта)
3) Если мутация есть только у одного родителя → `100%` (или `80% мутация / 20% пусто` если хочешь “сбрасываемость”)

**Запрет для MVP:**
- “случайный новый боевой эффект” как Wild roll. Это почти всегда ломает баланс и вызывает злость.

---

## 5) Strain (напряжение генома) — как сделать рост цены без экспоненты

### 5.1 Расчёт Strain

- `Strain = 3 * (#CombatMutationsActive) + 1 * (#CosmeticTraitsActive)`

Примеры:
- 2 боевые мутации + 1 трейт → `3*2 + 1*1 = 7`
- 1 боевая + 2 трейта → `3 + 2 = 5`

### 5.2 Доплата за сложность (Fee)

Рекомендуемая формула (простая и честная):
- `ExtraFeeDoses = ceil(Strain / 3)`
- Standard Breed платит `ExtraFee` в **Genome** (или в Doses, если ты хочешь “двухступенчатость”)
- Splice платит `ExtraFee` + `Core Sample`

> Не делай экспоненту. Она мгновенно превращает “долгий прогресс” в “я никогда это не соберу”.

---

## 6) Родители: удалять или нет?

### 6.1 Решение по умолчанию: **НЕ удалять**
Удаление родителей:
- убивает коллекционирование,
- создаёт страх экспериментировать,
- повышает rage quit.

### 6.2 Синк вместо удаления: Fertility Charges + Recycle

#### Fertility Charges
- У питомца есть `Fertility = 3..5` (балансится)
- Каждый бридинг тратит 1
- Восстановление:
    - **еда** (Biomass → Fertility Food)
    - или “длинный” пассивный реген (медленно)

#### Recycle / Extract (добровольная утилизация)
Игрок может разобрать ненужного питомца:
- получает `Genome + Shards (+ маленький шанс Style компонент)`
- это добровольный sink, который не бесит

---

## 7) Межрасовой бридинг (если хочешь и это реально нужно)

Не открывай межрасовой бридинг “бесплатно” в MVP — это взрывает матрицу контента.

Режим:
- **Cross-species** разрешён только через **Splice**
- Требует `Core Sample + дополнительный Genome`
- Результат:
    - Dominant Gene = выбранный/гарантированный
    - часть traits берётся по обычным правилам
    - “гибридные” traits/косметика — отдельный список (редкий, но не сила)

---

## 8) Баланс-ручки (то, чем ты реально будешь управлять)

- `P_single` для cosmetic trait наследования
- `P_none` (чтобы трейты не размножались без контроля)
- `FertilityCharges` и скорость восстановления
- `Shard→Serum` коэффициент (10→1 по умолчанию)
- `Strain weights` (3 для боевых, 1 для косметики)
- шанс/стоимость “Wild Cosmetic Roll” (если вводишь)

---

## 9) Пример (чтобы проверить, что система читается)

Parent A:
- Combat: [Poison, Shield]
- Cosmetic: [Neon Veins]
- Strain = 3*2 + 1 = 7

Parent B:
- Combat: [Poison, Lifesteal]
- Cosmetic: [—]
- Strain = 6

Ребёнок (Standard):
- Combat slot 1: Poison (100%, одинаково)
- Combat slot 2: Shield vs Lifesteal (50/50)
- Cosmetic: Neon Veins (P_single = 40%)
- Cost: Credits + Genome + ExtraFee (ceil(StrainChild/3))

## 10) Список “что обязательно показывать игроку” (иначе будет хейт)

- кто доминант и почему
- шансы по каждому слоту мутаций
- шансы по каждому cosmetic trait
- что именно выросло в цене из-за Strain
- сколько осталось Fertility Charges у родителей

## 11) Минимальный контент для MVP (чтобы не утонуть)

- Combat Mutations: 8–10
- Cosmetic Traits: 10–15
- Splice гарант: 3 варианта (trait / mutation slot / dominant gene)
- 2 режима рейда: Wave + Boss
- 1–2 ежедневных контракта + 1 недельный
