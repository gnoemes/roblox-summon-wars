# PROJECT: INCUBATOR ZERO - Mutate Pets. Clear Raids. (Roblox)

## 0) Elevator Pitch
Мультиплеерный хаб в стилистике милого sci-fi био-лаба + инстанс-рейды. Игроки разводят и усиливают “фэнтези-существ”, добавляя биомутации и кибернетику, и отправляют их в короткие PvE-рейды на арене. Бой: вид сверху (2.5D), питомец автоатакует, игрок командует перемещением (hold-to-move) и использует командные решения (стойки/приказы/расходники), ограниченные ресурсом.

---

## 1) Visual / Theme / Lore

### 1.1 Сеттинг
- Основная эстетика: **Bio-lab sci-fi**, чистые формы, неон-акценты, стекло/металл/биогель.
- Тон: **милый, плюшевый**, дружелюбный, без хоррора.
- Лор-рамка:
  - Игроки — исследователи/учёные из sci-fi цивилизации.
  - Открыта фэнтези-планета/мир с “местной фауной”.
  - Разведение и “улучшение” существ — популярное развлечение и соревновательный спорт.
  - Дальнейшее развитие: разделение апгрейдов на **биомутации** и **кибернетику** (две ветки прогрессии).

### 1.2 Арт-принципы (для top-down)
- Силуэт питомцев крупный, простой, читаемый при малом размере на экране.
- Минимум мелких деталей в моделях; “дороговизна” — через материалы и аккуратные акценты.
- Фон арены нейтральный, питомец/враги контрастные.
- Высокие props в аренах — только по краям, чтобы не перекрывать бой.

### 1.3 VFX-дисциплина
- VFX обслуживает читаемость:
  - телеграфы атак (круг/конус на земле),
  - статусы (иконка + тонкая аура),
  - проки мутаций — короткие, не спамящие.
- Отдельный слой косметики:
  - ауры/трейлы/альтернативные частицы.
- Переключатель качества эффектов: **Low/High VFX**.

## 2) Camera & Controls

### 2.1 Камера
- Бои: **2.5D top-down** (почти сверху, небольшой наклон 15–25°, фиксированный yaw).
- Хаб: **3rd person** по умолчанию.
- Станции в хабе (бридинг/апгрейды/контракты): переключение на “тактический” вид (изометрический/2.5D) только во время взаимодействия.

### 2.2 Управление в бою (Commander Mode)
- Питомец:
  - автоатака,
  - авто-таргет (настраиваемые приоритеты),
  - поведение меняется стойками и командами.
- Игрок:
  - **hold-to-move**: удержание ЛКМ/пальца обновляет целевую точку движения питомца.
  - Опция **Auto-move** (питомец сам перемещается между целями) — можно включить/выключить.
  - Доп. команды через очки командования (Command Points, CP).

## 3) Core Gameplay Loops

### 3.1 Core Loop (1–3 минуты)
1) Запуск рейда → волны врагов.
2) Командование перемещением + стойки + ситуативные команды.
3) Победа/поражение → награда (валюта/ресурсы/ДНК/ингредиенты).
4) Улучшение питомца / бридинг / подготовка → следующий рейд.

### 3.2 Meta Loop (дни/недели)
- Формирование “идеального” питомца:
  - сбор ресурсов,
  - бридинг (наследование/комбинации),
  - подбор мутаций/кибер-модулей под модификаторы рейдов.
- Открытие новых арен/боссов/модификаторов.
- Коллекционирование косметики.

## 4) Game Modes

### 4.1 Wave Arena (основной режим)
- Плоская арена, враги заходят с разных сторон.
- Длительность: 2–6 минут (настраиваемо).
- Цель: выживание/набор очков/достижение волны.
- Награды: базовая валюта + ресурсы под мутации/бридинг.

### 4.2 Boss Contracts (короткий бой с боссом)
- Длительность: 2–4 минуты.
- Босс с телеграфами и фазами; требует грамотного позиционирования и команд.
- Награды: ингредиенты для детерминированных апгрейдов/мутаций.

### 4.3 Modifiers System (контент-мультипликатор)
- Рейды получают модификаторы, которые меняют мету и билд-ценность:
  - усиление/ослабление типов урона,
  - ауры врагов (замедление/яд/щит),
  - лимиты лечения,
  - “элитные” волны,
  - повышенная агрессия/скорость мобов.
- Это основной способ добавлять контент без постоянного производства новых карт.

### 4.4 PvP (отложено)
- Приоритет: PvE.
- PvP при необходимости — асинхронное/ограниченное, без боевых расходников.

## 5) Pet System

### 5.1 Архетипы питомцев (MVP)
- 3 стартовых архетипа:
  - Tank (выживаемость/агро),
  - DPS (урон/проки),
  - Support (хил/баф/дебаф).

### 5.2 Stats (минимум)
- HP, Damage, Defense, AttackSpeed
- Доп. параметры (по мере роста):
  - Crit, Lifesteal, StatusPower, CooldownReduction.

### 5.3 Mutations & Cybernetics
- Мутации:
  - влияют на проки (яд, щит, цепь, вампиризм, отражение и т.п.)
  - дают “билд-направление”.
- Кибернетика:
  - модули поведения/поддержки (реакция на дебафы, улучшенная стойка, усиленные команды).
- В MVP: только мутации (кибер — позже или в виде косметики).

### 5.4 Slots
- 2 слота мутаций в MVP.
- Расширение слотов — через прогрессию и/или честные покупки удобства (не рандом).

## 6) Breeding / Genetics

### 6.1 Бридинг
- Формат: 2 → 1.
- Наследование:
  - часть мутаций/трейтов переносится по правилам,
  - возможны комбинации (если совместимые).
- Прозрачность:
  - игрок должен понимать вероятность/правила (не “чёрный ящик”).

### 6.2 Рандом и ограничения
- Рандомные исходы (мутации/комбо) существуют только:
  - через бридинг,
  - через внутриигровые механики с непокупаемой валютой.
- Запрещённая зона:
  - любые покупки, влияющие на вероятность/рандом, напрямую или косвенно.

## 7) Combat & Commanding

### 7.1 Бой
- Питомец автоатакует ближайшие/приоритетные цели.
- Угрозы врагов в основном мили; опасность через:
  - окружение,
  - зоны на земле,
  - ауры и дебафы.

### 7.2 Stances (Attack/Def/Balance)
- Attack: выше урон, агрессивный выбор целей.
- Def: выше выживаемость, осторожное позиционирование.
- Balance: средний режим.

### 7.3 Command Points (CP)
- CP — ресурс для команд.
- В бою игрок регулярно тратит CP на решения.

### 7.4 Команды (MVP набор)
- Focus Target (смена приоритета цели)
- Emergency Shield / Cleanse (реактивная защита)
- Burst Window (короткий буст, затем откат/усталость)

## 8) Items / Potions

### 8.1 Ограничение
- **2 зелья на бой** максимум.

### 8.2 Балансная цель
- Зелья ситуативные, не “налог на прохождение”.
- Дорогие по внутриигровой валюте, но реально доступные регулярно за счёт грамотной экономики.

### 8.3 Типы зелий (предпочтительно)
- маленький хил,
- снятие дебаффа,
- короткий щит,
- одноразовый “анти-смерть” (мягкий сейв).

- Большие зоны нажатия, минимальный текст, иконки.

## 11) Monetization & Compliance

### 11.1 Принципы
- Без paid-random механик.
- Всё, что продаётся за Robux, должно быть:
  - косметика,
  - удобство,
  - детерминированные эффекты (без влияния на вероятности).

### 11.2 Категории монетизации
1) Cosmetics:
   - скины питомцев, ауры, трейлы, VFX-варианты, рамки профиля.
2) Deterministic Mutations:
   - “еда/инъекция” за Robux, которая **гарантированно** даёт конкретную мутацию/модуль.
3) Convenience:
   - дополнительные слоты (питомцы/пресеты),
   - ускорение НЕ случайных процессов (например, таймеров удобства, если появятся).
4) Safety / Limited Utility:
   - ограниченные revive-токены для PvE (желательно в рамках дневного/недельного лимита),
   - без применения в PvP.

### 11.3 Нельзя
- продавать шанс/удачу/повышение вероятности мутаций,
- продавать внутриигровую валюту, если она используется для рандома/бридинга,
- продавать боевые “темпо-мультипликаторы” (например, CP regen x2) как основной товар.

## 13) Server / Place Architecture

### 13.1 Структура Experience
- Place A: Hub
  - публичные серверы (10–30 игроков)
  - социальная витрина, станции, подготовка
- Place B: Raid
  - приватные инстансы (1–4 игрока)
  - волны/боссы/модификаторы

### 13.2 Переходы
- Hub → Raid:
  - создаётся reserved server (access code)
  - телепорт пати в RaidPlace
- Raid → Hub:
  - выдача наград сервером рейда
  - телепорт обратно в Hub

### 13.3 Seamless ощущение (UX)
- Экран затемнения/подготовки
- Быстрая подгрузка ассетов
- Мгновенный спавн и старт таймера
- Возврат в хаб через короткий “debrief” экран

### 13.4 Изоляция
- В рейде изоляция полная (private instance).
- Внутрисерверный “инстансинг” в одном мире не используется как основная модель.

## 14) Roadmap

### MVP (P0, запуск)
- Hub (3rd person) + станции (тактический вид)
- Wave Arena: 1 арена
- Boss Contract: 1 босс
- 8–10 мутаций, 2 слота
- 3 стойки + CP + 2–3 команды
- Breeding 2→1
- 2 зелья на бой
- Базовая косметика
- Аналитика и минимальные KPI

### P1 (после первой валидации метрик)
- 10–20 модификаторов рейдов
- Daily/Weekly + streak
- 2–3 босса
- Расширение косметики и витрины
- Low/High VFX

### P2
- Ветка кибернетики
- Социальные системы (кланы/совместные контракты)
- PvP (если нужно) — ограниченно и без расходников

## 15) Key Risks & Mitigations

1) Риск: игра воспринимается как idle/симуляция
   - Митигация: постоянное командование move + частые CP-решения.

2) Риск: P2W восприятие из-за зелий/ревивов
   - Митигация: зелья ситуативные, лимиты, отсутствие темпо-мультипликаторов, честный фритуплей доступ.

3) Риск: контент быстро кончается
   - Митигация: модификаторы рейдов, короткие режимы, быстрая ротация.

4) Риск: комплаенс вокруг рандома
   - Митигация: никаких покупок, влияющих на шанс; внутриигровая рандом-валюта не покупаемая.
