## 1) Цель механики
Бридинг = основной путь:
- создавать новых питомцев (2 → 1),
- развивать **Genome Capacity** поколениями,
- наследовать/эволюционировать мутации,
- делать Fusion “крафт мутаций”.

---

## 2) Core правила (вертикальный срез)
- Бридинг всегда “2 родителя → 1 ребёнок”.
- Родители **не удаляются**.
- Результат бридинга прозрачен: игрок видит шансы/правила до подтверждения.

---

## 3) Наследование Genome Capacity

### 3.1 Определения
- `CapA`, `CapB` — Genome Capacity родителей.
- `CapMax` — потолок: 10.

### 3.2 Правило (текущее)
1) Если `CapA == CapB == N` и `N < CapMax`:
   - 50% → `N+1`
   - 50% → `N`

2) Если пара вида `{N, N+1}`:
   - 33% → `N+1`
   - 67% → `N`

3) Если разница больше 1:
   - поведение фиксируется позже (варианты: запрет пары / привести к правилу для разницы 1 / отдельная формула)

---

## 4) Наследование мутаций (без списков контента)

### 4.1 Visual Mutations
- Visual мутации не расходуют Capacity.
- Наследуются по каналам (Head/Torso/FullBody/Aura/Accessory/Color).
- Конфликты каналов применяются по правилам `Mutations_System`.

### 4.2 Combat Mutations
- Combat мутации ограничены Genome Capacity ребёнка.
- Базовый принцип:
  - если мутация есть у обоих родителей (одинаковая) → 100% наследование
  - если мутация есть только у одного → участвует в пуле кандидатов

Так как “слотов” нет (есть Capacity), конкретный алгоритм отбора формализуется отдельно:
- сначала фиксируются “общие” мутации,
- затем добираются кандидаты до заполнения Capacity.

---

## 5) Fusion (крафт мутаций)

### 5.1 Место Fusion
Fusion может происходить:
- как часть результата бридинга (если пара мутаций соответствует рецепту),
- или как отдельная операция в лаборатории (решение фиксируется позже).

### 5.2 Ограничение
- Fusion возможен только по курируемым рецептам (например 10).

### 5.3 Принцип стоимости
- Combat+Combat → чаще компрессия по Capacity.
- Visual+Visual → может давать Visual-комплект (занимает другой канал).
- Visual+Visual → Combat допускается только для явно разрешённых рецептов и обычно дороже по Capacity.

---

## 6) Межвидовой бридинг (рамка)
Чтобы результат был предсказуем:
- В MVP допускаются “группы совместимости” (breed только внутри группы).
- Cross-species (если будет) — отдельными правилами/операциями, чтобы избегать неопределённого результата.

---

## 7) MVP vs Post-MVP

### MVP
- 2→1 breeding
- Наследование Genome Capacity по правилам 50% (N+N) и 33% (N+N+1)
- Наследование Visual мутаций по каналам
- Combat мутации через Genome Capacity
- Fusion через ограниченный набор рецептов

### Post-MVP
- расширение fusion/рецептов
- расширение правил межвидового бридинга (если принято)
- дополнительные механики контроля наследования (если понадобятся)
