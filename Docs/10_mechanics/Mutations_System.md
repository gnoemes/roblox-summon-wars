## 1) Цель системы
Дать игроку **глубину билдов** через мутации, не превращая прогресс в “казино ради силы”.

Принципы:
- Боевая сила ограничена **Genome Capacity** (бюджетом), а не жёстким числом слотов.
- Визуальные мутации ограничены **визуальными каналами**, и не расходуют Capacity.
- Мутации должны синергировать с командованием (Dash / Focus / Hold / Stances).

---

## 2) Базовые сущности

### 2.1 Genome Capacity
- У каждого питомца есть параметр **Genome Capacity** — “бюджет” для боевых мутаций.
- Стартовое значение у покупного питомца: **4**.
- Долгосрочный потолок (эндгейм): **10**.

**Как используется:**
- Каждая боевая мутация имеет стоимость `MutationCost`.
- Сумма стоимостей активных боевых мутаций не может превышать Genome Capacity.

### 2.2 Типы мутаций
1) **Combat Mutations** (расходуют Capacity)
   - пассивы, проки, триггеры, модификаторы команд/скиллов.

2) **Visual Mutations** (НЕ расходуют Capacity)
   - визуальные изменения питомца (ауры, части тела, аксессуары, палитра и т.п.).

---

## 3) Visual Mutations — слоты/каналы

### 3.1 Каналы (по 1 активной мутации на канал)
- **Head**
- **Torso**
- **Full Body** (конфликтует с Head+Torso)
- **Aura**
- **Accessory** (например рога/шипы)
- **Color/Palette**

### 3.2 Правила конфликтов
- Если активирован **Full Body**, то **Head** и **Torso** недоступны (и наоборот).
- Aura всегда одна.
- Accessory всегда один.

---

## 4) Combat Mutations — категории (механика, без списка контента)
Combat мутации делятся на семейства:
- **Proc**: onHit / onKill / onDamageTaken / periodic
- **Command Augment**: onDash / whileHold / onFocus / onStanceChange
- **Skill Modifier**: меняют поведение одного конкретного skill (например делают его тактическим)

Каждая мутация должна:
- быть читаемой по триггерам,
- иметь понятную стоимость Capacity,
- не требовать “умного AI распознавания” для MVP (только простые события).

---

## 5) Marionette (марионетка) — тактический случайный skill

### 5.1 Идея
Боевая мутация **Marionette** делает так, что в рейде **один случайный skill** питомца становится **Tactical** (ручной), без настройки в хабе.

### 5.2 Стоимость
- `MutationCost = 2` (consumes Genome Capacity)

### 5.3 Когда выбирается тактический skill
Чтобы избежать “быстрых перезапусков ради реролла”:
- Тактический skill назначается **после достижения milestone 25% волн** в Wave Arena.
- Для Boss Contract (если он есть): момент назначения фиксируется позже.

### 5.4 TacticalEligible пул
- В пул выбора попадают только skills с флагом `TacticalEligible = true`.
- Если eligible skills нет — Marionette не активируется в этом рейде.

### 5.5 UX
- На HUD у выбранного skill появляется маркер “Tactical”.
- **Long-press** по кнопке переводит этот skill в **Auto** до конца рейда.
- Tactical назначается **ровно один** на рейд (не накапливается).

---

## 6) Fusion (крафт) мутаций — ограниченный набор рецептов

### 6.1 Общая идея
Fusion объединяет две мутации в одну “комбинированную”.
Цель:
- ощущение “эволюции” и крафта,
- система остаётся контролируемой.

### 6.2 Ограничение
- Fusion поддерживает **курируемый список** рецептов (например 10).
- Никакой “любая с любой”.

### 6.3 Стоимость и баланс (правило)
- **Combat + Combat → Combat**: чаще “компрессия” Capacity (дешевле, чем сумма), но не бесплатная.
- **Visual + Visual → Combat** (если допускается конкретным рецептом): это “апгрейд” и стоит дороже по Capacity.

---

## 7) Требования к интерфейсу (минимум)
UI должен показывать:
- текущий Genome Capacity и занятый Capacity (сумма)
- активные Combat мутации с `MutationCost` и триггерами
- Visual мутации по каналам
- для Marionette: какой skill стал Tactical и как вернуть Auto (long-press)
- для Fusion: ингредиенты → результат → изменения по Capacity/каналам

---

## 8) MVP vs Post-MVP

### MVP
- Genome Capacity (start=4, рост через бридинг)
- Visual channels (6)
- Combat мутации через Capacity
- Marionette (1 tactical skill/рейд, выбор после 25% волн, long-press → auto)
- Fusion: ограниченный список рецептов

### Post-MVP
- расширение Fusion-рецептов
- более глубокие триггеры (telegraphs/danger zones)
- дополнительные типы мутаций (если появятся кибернетика/модули)
