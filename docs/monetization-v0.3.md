# Monetization v0.3

**Status:** Fixed baseline  
**Last updated:** 2026-03-08

---

## 1. Purpose

This document updates the monetization baseline after the stand/guidance integration pass.

It covers:
- what is sold;
- what is accelerated;
- which systems are intended to drive revenue;
- which monetization directions are acceptable;
- what should be avoided.

This remains a product baseline, not a final live-ops pricing guarantee.

---

## 2. Monetization Philosophy

The project is intended to make money.

The intended model remains:
- accelerate progression;
- sell convenience;
- sell storage;
- sell packs;
- later sell cosmetics / prestige.

The intended model is still **not**:
- direct pay-for-win combat outcomes;
- forced spending to access normal play;
- mandatory paid consumables for normal play;
- monetized raid victories;
- hard paywalls on core progression.

### Core principle
Players should feel:
- spending is useful;
- spending is tempting;
- spending saves time;
- spending adds convenience;
- spending can improve visible status and presentation.

Players should not feel:
- spending is required to play properly;
- free progression is fake;
- the shop is stronger than the game.

## 3. Main Monetization Layers

| Layer | What is sold | Main role | Toxicity risk | Priority |
|---|---|---|---:|---|
| **Soft Currency** | currency packs for Robux | universal accelerator | 6/10 | P0 |
| **Packs** | low / mid / high card packs | core revenue and impulse purchases | 8/10 | P0 |
| **Storage** | fighter slots / chest slots | natural pressure relief + convenience | 4/10 | P0 |
| **Boosters** | XP / basic resource / currency boosters | soft progression acceleration | 6/10 | P0 |
| **Subscription** | storage + QoL + mild XP buff | recurring revenue | 5/10 | P0.5 |
| **QoL Sales** | presets, bulk tools, sorting | engaged-user convenience | 3/10 | P1 |
| **Cosmetics / Prestige** | stand visuals, frames, effects, identity layers | long-term healthy monetization | 2/10 | P1/P2 |

---

## 4. Red Lines

The following should not be aggressively monetized:
- direct raid victory;
- paid mandatory raid entry as the norm;
- rare or unique loot chance boosters;
- Legendary chance boosters;
- guaranteed progression wins through spending only;
- paid mandatory consumables for normal play;
- raw resource slot pain as a baseline monetization weapon.

---

## 5. Soft Currency

Baseline gameplay-facing assumption:
**1 Robux = 10 soft currency**

Soft currency is used to:
- buy packs;
- pay some services;
- support part of crafting or convenience later.

Soft currency should accelerate the loop, not replace gameplay entirely.

| Pack | Soft Currency |
|---|---:|
| **Low Pack** | **400–500** |

### Class Packs
- class packs cost more than random equivalents
- class packs do not include Legendary fighters
- random packs sell ceiling; class packs sell control

## 7. Storage

Storage pressure remains a natural monetization pillar in this type of game.

### Important rule
Baseline storage must stay usable.
Storage monetization should relieve friction, not create instant resentment.

### Important clarification
In baseline design, storage should mean:
- **fighter roster slots**
- **consumable chest slots**

It should **not** mean:
- forcing raw resource slot pain from minute one.

### Fighter Storage
| Purchase | Baseline Price |
|---|---:|
| **+5 fighter slots** | **79 Robux** |
| **+10 fighter slots** | **149 Robux** |

### Chest Storage
| Purchase | Baseline Price |

| **+25 chest slots** | **149 Robux** |

### Storage Bundle
| Bundle | Baseline Price |
|---|---:|
| **Roster + Chest Bundle** | **249 Robux** |

## 8. Boosters

Boosters remain a monetized acceleration layer separate from crafting and combat consumables.

### Fixed design rule
Boosters may affect:
- XP;
- basic resources;
- base currency.

Boosters should **not** affect:
- unique raid loot;
- Legendary chance;
- Epic/Legendary card chance;
- unique endgame drop tables.

### Booster format rule
The preferred model remains:
- **wins / clears based duration**
not real-time countdown pressure.

---

## 9. Subscription

Subscription remains a P0.5 layer focused on:
- storage relief;
- QoL;
- mild safe acceleration.

It should not become:
- a hidden necessity to play normally;
- a raid win pass;
- a combat power requirement.

---

## 10. Cosmetics / Prestige

Later monetization should increasingly lean on:
- stand visuals;
- board styles;
- aura;
- banners;
- titles;
- squad display flair.

This direction is stronger now because the project uses:
- public stands;
- visible statistics;
- social comparison;
- prestige-friendly shell design.

---

## 11. Design Intent Summary

Monetization should make the product healthier, not dirtier.

The ideal result is:
- free players can progress and enjoy the game;
- spenders move faster or look better;
- prestige has visible value;
- storage relieves friction without manufacturing fake pain;
- the shop supports the game instead of replacing it.
