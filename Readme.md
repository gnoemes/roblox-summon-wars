# IncubatorZero — Dev Scripts (Rojo/Wally/Luau)

Этот репозиторий использует файловый воркфлоу (IDE → файлы → Rojo → Roblox Studio), а не редактирование кода внутри Studio.

Ниже описаны скрипты в `./scripts`, их назначение и когда их запускать.

---

## Основные скрипты

### 1) `scripts/DevStart.ps1` — старт дев-сессии (Rojo Serve + Sourcemap Watch)
**Что делает:**
- Поднимает **Rojo serve** для live-синка проекта в Roblox Studio.
- Поднимает **Rojo sourcemap --watch** для генерации/обновления `sourcemap.json` (нужно для Luau LSP/IDE).
- Логи пишет в `.dev/logs/`.
- Сохраняет PID процессов в `.dev/pids.json`, чтобы их можно было корректно остановить.

**Когда вызывать:**
- Каждый раз, когда ты начинаешь рабочую сессию и хочешь проверять код через Roblox Studio.
- После перезагрузки ПК/закрытия Rojo процессов.
- После смены структуры проекта / `default.project.json` (чтобы sourcemap обновлялся корректно).

**Как использовать:**
1) Запусти `scripts/DevStart.ps1`
2) Открой Roblox Studio → Rojo plugin → Connect
3) Дальше редактируй код в IDE и проверяй через Play/Stop в Studio.

---

### 2) `scripts/DevStop.ps1` — остановка дев-сессии
**Что делает:**
- Останавливает процессы, поднятые `DevStart.ps1` (Rojo serve и sourcemap watch) по PID из `.dev/pids.json`.
- Удаляет `.dev/pids.json`.

**Когда вызывать:**
- В конце рабочего дня/сессии.
- Перед перезапуском `DevStart.ps1`, если процессы уже запущены и зависли.
- Если нужно освободить порты/ресурсы (например, Rojo не даёт повторно запуститься).

---

### 3) `scripts/Check.ps1` — проверка перед коммитом (format + lint + build)
**Что делает:**
- Запускает форматирование (через `Fmt.ps1`).
- Запускает линт (через `Lint.ps1`).
- Делает `rojo build` в `.dev/build.rbxm` (быстрая проверка, что проект валиден и собирается).

**Когда вызывать:**
- Перед каждым коммитом/пушем.
- Перед тем как просить ревью/сливать изменения.
- После крупных изменений структуры/зависимостей, чтобы убедиться, что проект не сломан.

---

## Дополнительные скрипты

### `scripts/Deps.ps1` — установка/обновление зависимостей (Wally)
**Что делает:**
- Выполняет `mise install` (ставит инструменты из `mise.toml`).
- Выполняет `wally install` (обновляет папку `Packages/`).

**Когда вызывать:**
- Когда изменил `wally.toml` или `wally.lock`.
- После `git pull`, если в репозитории изменились зависимости.
- Если папка `Packages/` отсутствует или повреждена.

---

### `scripts/Fmt.ps1` — форматирование кода (StyLua)
**Что делает:**
- Запускает `stylua` по исходникам (обычно `src/`).

**Когда вызывать:**
- Перед коммитом (обычно через `Check.ps1`).
- После больших правок, чтобы привести код к единому стилю.

**Зависит от:**
- `.stylua.toml` в корне проекта (правила форматирования).

---

### `scripts/Lint.ps1` — статический анализ (Selene)
**Что делает:**
- Запускает `selene` по исходникам (обычно `src/`).

**Когда вызывать:**
- Перед коммитом (обычно через `Check.ps1`).
- Когда хочется поймать ошибки раннее (неиспользуемые переменные, опечатки, подозрительные конструкции).

**Зависит от:**
- `selene.toml` в корне проекта.

---

### `scripts/TailLogs.ps1` — просмотр логов дев-процессов
**Что делает:**
- Показывает/подписывается на логи из `.dev/logs/` (полезно, если Rojo/LSP что-то ругается).

**Когда вызывать:**
- Если `DevStart.ps1` отработал, но Rojo/сourcemap не поднялись.
- Если в Studio не коннектится Rojo plugin или синк не работает.

---

## Рекомендованный ежедневный флоу

### Начало сессии
1) (Опционально) `scripts/Deps.ps1` — если были изменения в зависимостях или после `git pull`.
2) `scripts/DevStart.ps1`
3) Roblox Studio → Rojo plugin → Connect
4) Разработка в IDE → Play/Stop в Studio для проверки.

### Перед коммитом
1) `scripts/Check.ps1`
2) `git commit` / `git push`

### Конец сессии
1) `scripts/DevStop.ps1`

---

## Где что хранится

- Исходники: `src/`
- Зависимости (генерируется): `Packages/` *(не коммитится)*
- Sourcemap (генерируется): `sourcemap.json` *(не коммитится)*
- Артефакты и логи (генерируется): `.dev/` *(не коммитится)*
- Конфиги:
    - `default.project.json` — Rojo project
    - `wally.toml` / `wally.lock` — зависимости
    - `.stylua.toml` — правила форматирования
    - `selene.toml` — правила линтинга
